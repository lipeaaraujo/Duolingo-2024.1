    Priorização das Histórias de Usuário#

    Neste documento será apresentada a tabela de priorização das histórias de usuário, gerada a partir do documento de Backlog. Além disso, foram levantados os critérios de aceitação e o nível de dificuldade de execução.

    Tabela 01 - Priorização das Histórias de Usuário#

    ID Critérios de Aceitação Prioridade Dificuldade
    US01 - [ ] O perfil deve permitir personalizar dados como nome, nome de usuário, avatar, telefone
    - [ ] As mudanças devem ser refletidas imediatamente no perfil

    		Alta Média
    US02 - [ ] O perfil deve apresentar informações importantes como dias de ofensiva, total de XP, divisão atual, troféis, amigos, seguidores e seguindo, cursos, conquistas
    - [ ] O layout do perfil deve ser simples e organizado
    - [ ] As informações do perfil devem sempre estar atualizadas    		 Média Média
    US03 - [ ] O usuário deve poder enviar e receber solicitações de amizade
    - [ ] O usuário deve poder compartilhar seu perfil e progresso em outras redes e por QR code
    		Média Média
    US04 - [ ] Visualizar o perfil de um amigo
    - [ ] Participar de missões de amigos, competir nas ligas e entre amigos
    - [ ] Enviar presentes por meio das missões de amigos
    - [ ] Compartilhar notificações de progresso e interagir com as notificações    		 Média Alta
    US05 - [ ] Receber sugestões com base nos amigos, interesses e interações
    - [ ] Poder visualizar o perfil do usuário sugerido
    - [ ] As sugestões devem estar visíveis na interface de perfil    		 Baixa Média
    US06 - [ ] O usuário deve poder cadastrar conta com email e senha
    - [ ] O usuário deve ser capaz de visualizar e editar as informações pessoais
    - [ ] O usuário deve ser capaz de excluir a conta permanentemente    		 Alta Média
    US07 - [ ] O usuário deve ser capaz de acessar utilizando outras plataformas como Google e Facebook
    - [ ] O usuário deve capaz de acessar usando as informações de uma conta registrada
    - [ ] O usuário deve ser capaz de acessar em qualquer dispositivo compatível com o app    		 Alta Média
    US08 - [ ] O usuário deve ser capaz de sair da conta com segurança
    - [ ] Todos os tokens da sessão devem ser removidos ou inutilizados    		 Alta Baixa
    US09 - [ ] O usuário deve ser capaz de acessar a mesma conta em dispositivos diferentes
    - [ ] Os dados devem estar constantemente sincronizados
    		Média Média
    US10 - [ ] O usuário deve ser capaz de salvar e excluir contas do mesmo dispositivo
    - [ ] O usuário deve ser capaz de alternar entre as contas salvas
    		Média Média
    US11 - [ ] O aplicativo deve enviar notificações de lembretes diários, atualizações sobre o progresso na liga, as novidades do aplicativo e atualizações de amigos
    - [ ] O aplicativo deve exibir as notificações enviadas    		 Média Média
    US12 - [ ] O usuário deve ser capaz de ativar ou desativar individualmente as notificações de lembretes, sobre a liga, novidades e amigos
    - [ ] O usuário deve ser capaz de definir a frequência dos lembretes e o horário para receber
    O usuário deve ser capaz de ativar ou desativar as notificações sobre novos seguidores e atividade dos amigos    		 Média Baixa
    US13 - [ ] As lições devem ser interativas, curtas e objetivas
    - [ ] As questões devem abranger exercícios de fala, leitura, escrita e escuta
    - [ ] O conteúdo deve ser confiável e verificado
    - [ ] O usuário deve ser capaz de realizar uma lição específica de seu conteúdo
    - [ ] Ao concluir a lição o progresso do usuário deve ser atualizado
    - [ ] Um novo vocabulário deve ser destacado
    		Alta Alta
    US14 - [ ] O usuário deve ser capaz de fazer uma lição de revisão após concluir um nível
    - [ ] O usuário deve ser capaz de refazer as questões incorretas durante uma lição
    - [ ] O usuário deve ser capaz de revisar as histórias abordadas
    - [ ] O usuário deve ser capaz de fazer uma lição de revisão da unidade    		 Alta Média
    US15 - [ ] O usuário deve ser capaz de avançar em uma unidade ou seção realizando um teste
    - [ ] O conteúdo do teste deve abranger todo o conteúdo da unidade ou seção
    - [ ] O teste deve oferecer recompensas maiores    		 Alta Baixa
    US16 - [ ] O feedback deve informar se o usuário acertou ou errou a questão, e no segundo caso apresentar a resposta correta
    - [ ] O feedback deve ser imediato após a realização da questão
    - [ ] As mensagens de feedback devem ser claras e objetivas    		 Alta Média
    US17 - [ ] O usuário deve receber a porcentagem de acertos, tempo gasto e a recompensa obtida
    - [ ] A mensagem de desempenho deve aparecer logo após completar uma lição
    - [ ] A mensagem de desempenho deve ser clara e objetiva    		 Média Média
    US18 - [ ] O usuário deve ser capaz de visualizar a Trilha de Progresso, com as lições já realizadas e as próximas lições
    - [ ] O usuário deve ser capaz de visualizar as unidades finalizadas em uma seção
    - [ ] O layout da trilha deve ser claro e organizado    		 Alta Média
    US19 - [ ] O usuário deve ser capaz de visualizar e realizar as histórias completadas
    - [ ] As coleções devem estar constantemente atualizadas
    - [ ] O usuários deve ser capaz de visualizar os erros mais recentes
    - [ ] O usuário deve ser capaz de visualizar o número de palavras aprendidas    		 Média Baixa
    US20 - [ ] As conquistas devem ser acessíveis na interface de perfil
    - [ ] O usuário deve ser capaz de compartilhar suas conquistas
    - [ ] As conquistas devem estar divididas em recordes pessoais e triunfos    		 Média Baixa
    US21 - [ ] O usuário deve ser capaz de visualizar a lista de cursos disponíveis
    - [ ] O usuário deve ser capaz de escolher um curso e iniciar o aprendizado
    - [ ] O curso escolhido deve ser configurado como o curso ativo na interface do aplicativo
    - [ ] O usuário deve responder um questionário de nivelamento antes de começar    		 Alta Média
    US22 - [ ] O usuário deve ser capaz de definir metas diárias ou semanais de aprendizado
    - [ ] O usuário deve ser capaz de definir o motivo para aprender o idioma    		 Alta Baixa
    US23 - [ ] O usuário deve ser capaz de definir o tempo disponível para estudo diário
    - [ ] O plano de estudo deve se ajustar automaticamente com base no tempo disponível definido    		 Média Baixa
    US24 - [ ] O usuário deve ser capaz de adiar os exercícios de fala e escuta
    - [ ] O usuário deve ser capaz alterar a velocidade de reprodução de exercíos de escuta
    - [ ] As alterações devem ser refletidas nos exercícios    		 Média Média
    US25 - [ ] O usuário deve ser capaz de competir nas ligas com os amigos
    - [ ] O progresso nas ligas deve ser apresentado para ambos participantes    		 Média Baixa
    US26 - [ ] O usuário deve ser capaz de participar de ligas semanais automaticamente
    - [ ] O progresso na liga deve ser atualizado em tempo real e apresentado no ranking
    - [ ] O usuário deve receber notificações de mudança de posição no ranking    		 Alta Alta
    US27 - [ ] O usuário deve ser capaz de visualizar e completar missões diárias e semanais
    - [ ] O progresso nas missões deve ser apresentado em tempo real
    - [ ] As recompensas pelas missões devem ser entregues automaticamente    		 Média Média
    US28 - [ ] O usuário deve ser capaz de visualizar e participar de missões em conjunto com amigos
    - [ ] O progresso nas missões de amigos deve ser atualizado e compartilhado com todos os participantes
    - [ ] Recompensas colaborativas devem ser entregues ao completar as missões
    [ ] - O amigo deve ser notificado da conclusão da missão    		 Média Média
    US29 - [ ] O usuário deve ser capaz de utilizar cristais para adquirir itens na loja do aplicativo
    - [ ] A loja deve ter itens como baús, bloqueios de ofensiva, estica tempo e cristais disponíveis
    - [ ] O inventário do usuário deve ser atualizado automaticamente após a compra
    - [ ] Os itens adquiridos devem estar disponíveis imediatamente para uso    		 Média Média
    US30 - [ ] O usuário deve ser capaz de comprar itens utilizando dinheiro real por meio de diferentes métodos de pagamento
    - [ ] A segurança das transações deve ser garantida
    - [ ] Itens premium adquiridos devem ser refletidos no inventário imediatamente    		 Média Alta
    US31 - [ ] O usuário deve ser capaz de recarregar as vidas após esgotá-las
    - [ ] O tempo de espera para recarregar deve ser exibido claramente
    - [ ] O usuário deve ser capaz de usar cristais recarregar vidas    		 Alta Média
    US32 - [ ] O usuário deve receber XP ao completar lições e missões
    - [ ] O XP deve ser automaticamente adicionado ao total acumulado
    - [ ] O progresso do usuário deve ser exibido no perfil e atualizado em tempo real    		 Alta Média
    US33 - [ ] O usuário deve receber baús contendo recompensas aleatórias ao completar certos desafios
    - [ ] O conteúdo dos baús deve ser variado e exibido de maneira surpreendente
    - [ ] As recompensas dos baús devem ser entregues automaticamente ao usuário    		 Média Baixa
    US34 - [ ] O usuário deve receber troféus ao completar conquistas importantes no aplicativo
    - [ ] Os troféus devem ser exibidos no perfil e podem ser compartilhados com amigos
    - [ ] O usuário deve ser notificado ao conquistar novos troféus
    - [ ] As conquistas devem estar constantemente atualizadas    		 Baixa Média
    US35 - [ ] O usuário deve ser recompensado ao completar desafios de ofensiva
    - [ ] As recompensas devem ser entregues automaticamente ao completar os desafios
    - [ ] O progresso do desafio deve ser exibido em tempo real    		 Média Baixa
    US36 - [ ] O usuário deve ser capaz de utilizar um item de bloqueio de ofensiva para evitar perder uma ofensiva
    - [ ] O bloqueio deve ser ativado automaticamente ao perder um dia de estudo
    - [ ] O progresso do bloqueio deve ser exibido no perfil    		 Média Baixa
    US37 - [ ] O usuário deve ser incentivado a manter uma ofensiva pessoal de estudo diário
    - [ ] O aplicativo deve monitorar o progresso diário e enviar lembretes da ofensiva
    - [ ] O progresso da ofensiva deve ser exibido de maneira clara e motivadora    		 Alta Média
    US38 - [ ] O usuário deve ser capaz de compartilhar sua ofensiva com amigos e manter um grupo motivado
    - [ ] As ofensivas compartilhadas devem exibir o progresso de todos os participantes
    - [ ] Recompensas devem ser entregues ao completar uma ofensiva em grupo
    - [ ] Os participantes devem receber notificações sobre a ofensiva compartilhada    		 Média Média
    US39 - [ ] A navegação pelo aplicativo deve ser simples e intuitiva
    - [ ] O usuário deve ser capaz de acessar todas as funcionalidades de maneira eficiente
    - [ ] A interface deve ser organizada e evitar confusão para o usuário    		 Alta Média
    US40 - [ ] O conteúdo deve ser adequado para diferentes idades
    - [ ] O aplicativo deve ser facilmente utilizável por crianças e idosos    		 Baixa Baixa
    US41 - [ ] O usuário deve ser capaz de acessar a maioria das funcionalidades do aplicativo sem custos adicionais
    - [ ] As funcionalidades gratuitas devem ser claramente separadas das pagas
    - [ ] O aplicativo deve incentivar o uso contínuo sem obrigatoriedade de assinatura premium    		 Alta Baixa
    US42 - [ ] O usuário deve ser capaz de acessar o aplicativo offline para realizar lições e missões
    - [ ] O progresso offline deve ser sincronizado automaticamente ao reconectar à internet
    - [ ] A interface offline deve ser tão funcional quanto a online    		 Média Média
    US43 - [ ] O usuário deve ser capaz de alterar o idioma do aplicativo a qualquer momento
    - [ ] O aplicativo deve ser traduzido de maneira correta e contextualizada
    - [ ] Todos os menus, mensagens e notificações devem ser traduzidos adequadamente    		 Alta Baixa
    US44 - [ ] O usuário deve ser capaz de alterar as preferências de experiência no aplicativo, como volume, mensagens, exercícios, notificações e visual
    - [ ] As preferências alteradas devem ser aplicadas imediatamente
    - [ ] As alterações devem ser refletidas na interface de maneira clara e funcional    		 Média Média
    US45 - [ ] O usuário deve ser capaz de reportar problemas técnicos ou de usabilidade no aplicativo
    - [ ] O processo de reporte deve ser simples e rápido
    - [ ] O suporte deve reconhecer o reporte e dar um retorno ao usuário    		 Alta Média
    US46 - [ ] O usuário deve ser capaz de acessar a Central de Ajuda para tirar dúvidas ou resolver problemas técnicos
    - [ ] A Central de Ajuda deve conter soluções para os problemas mais comuns
    - [ ] O usuário deve ser capaz de navegar facilmente pelas opções da Central de Ajuda    		 Alta Baixa

    Autores: Felipe Amorim de Araújo, Raquel Ferreira Andrade, 2024
